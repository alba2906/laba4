## Цель работы

Изучить теоретические основы регулярных выражений и их применение для поиска подстрок в тексте.
Освоить навыки работы с библиотекой регулярных выражений, а также интеграцию алгоритмов поиска в графический интерфейс приложения.

---

## Постановка задачи

Необходимо разработать модуль поиска подстрок с использованием регулярных выражений и встроить его в существующее приложение — текстовый редактор.

### Требования к программе

Программа должна:

* интегрировать поиск в интерфейс текстового редактора;
* иметь выбор типа поиска;
* запускать анализ по кнопке;
* выводить результаты в таблицу;
* отображать:

  * найденную подстроку;
  * номер строки;
  * номер символа;
  * длину;
* подсвечивать найденный фрагмент в тексте;
* отображать количество найденных совпадений;
* очищать предыдущие результаты;
* обрабатывать пустой ввод.

# 1. Поиск HEX-кодов

## Регулярное выражение

```regex
(?<![A-Za-z0-9_])#[0-9A-Fa-f]{6}(?![A-Za-z0-9_])
```

## Подробное объяснение

| Элемент       | Значение                     |
| ------------- | ---------------------------- |
| `(?<!...)`    | отрицательная проверка слева |
| `#`           | начало HEX                   |
| `[0-9A-Fa-f]` | допустимые символы           |
| `{6}`         | ровно 6 символов             |
| `(?!...)`     | проверка справа              |

## Примеры (должны находиться)

* `#FFFFFF`
* `#123ABC`
* `#00ff00`

## Примеры (не должны находиться)

* `#FFF`
* `#12345`
* `#1234567`
* `#ZZZZZZ`
* `text#AA11CC`

# 2. Поиск акронимов (Regex)

## Регулярное выражение

```regex
(?<![#A-Za-z0-9_])(?:[A-Z]{2,}|(?:[A-Z]\.){2,}[A-Z]\.)(?![A-Za-z0-9_])
```

## Подробное объяснение

| Элемент           | Значение                     |
| ----------------- | ---------------------------- |
| `[A-Z]{2,}`       | 2+ заглавных букв            |
| `(?: ... )`       | группа                       |
| `(?:[A-Z]\.){2,}` | буква + точка минимум 2 раза |
| `[A-Z]\.`         | последняя буква + точка      |
| `(?<!...)`        | защита от вхождения          |
| `(?!...)`         | защита справа                |

## Примеры (должны находиться)

* `NASA`
* `FBI`
* `EU`
* `N.B.A.`
* `U.S.A.`

## Примеры (не должны находиться)

* `NaSa`
* `A`
* `N.B`
* `textNASA`

---

# 3. Поиск HTML-тегов

## Регулярное выражение

```regex
<\s*[A-Za-z][A-Za-z0-9]*(?:\s+[A-Za-z_:][A-Za-z0-9_:\-\.]*\s*=\s*(?:"[^"]*"|'[^']*'))+\s*/?\s*>
```

## Объяснение

* `<` — начало
* имя тега
* обязательные атрибуты
* значения в кавычках

## Примеры (должны находиться)

* `<div class="main">`
* `<img src="logo.png">`
* `<a href="link">`

## Примеры (не должны находиться)

* `<div>`
* `<a href=>`
* `<span class=main>`

5. Продолжение поиска
